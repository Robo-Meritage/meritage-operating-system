# Meritage Operating System

> The single source of truth for how Meritage runs — strategy, operations, people, and decisions, all in one place.
>
> ## What This Is
>
> This repository is the **Meritage Operating System (MOS)** — a living, version-controlled knowledge base that captures how the company thinks, operates, and makes decisions. Every document lives here in Markdown so it is readable by humans, rendered by GitHub, and ingested by AI tools.
>
> ## Why It Exists
>
> - **AI-ready** — Markdown is the native format for language models. Every file here can be fed directly to Claude, GPT, or any other AI assistant.
> - - **Version-controlled** — Every change is tracked. You can see who changed what, when, and why.
>   - - **Searchable** — Plain text is the most searchable format that exists.
>     - - **Portable** — No proprietary formats. No locked-in tools. Works everywhere.
>

> ## Naming Convention
>
> All files use lowercase, hyphen-separated names that describe their content clearly:
>
> | Instead of | Use |
> |---|---|
> | `proposal.docx` | `affiliate-program.md` |
> | `Call Notes Final v2.docx` | `management-call-playbook.md` |
> | `Fee Structure.xlsx` | `reverse-lehman-formula.md` |
> | `Weekly Sync.docx` | `weekly-operating-rhythm.md` |
> | `New Hire.docx` | `employee-onboarding.md` |
>
> ## Getting Started
>
> 1. Browse the `docs/` folder to find what you need.
> 2. 2. Use GitHub search (`/`) to find any document by keyword.
>    3. 3. To add a document, create a new `.md` file in the appropriate folder and open a PR.
>       4. 4. Follow the conventions in [CONTRIBUTING.md](CONTRIBUTING.md).
