# Meritage Capability Maturity Model

> This document tracks the operational maturity of each advisory capability within the Meritage Operating System. It answers the question: *Not what files exist — but how capable is the firm?*
>
> ---
>
> ## Maturity Scale
>
> | Level | Label | Description |
> |---|---|---|
> | 1 | **Scaffold** | Structure exists. No substantive content. Cannot be used operationally. |
> | 2 | **Planned** | Scope and intent defined. Key decisions documented. Not yet executable. |
> | 3 | **Functional** | Core process documented. Can be executed by an experienced advisor. Gaps remain. |
> | 4 | **Operational** | Fully documented. Standards, templates, and AI assistance in place. Usable by any advisor. |
> | 5 | **Institutionalized** | Lessons learned integrated. Tested in real engagements. Continuously improved. |

> ## How to Use This Document
>
> This tracker is a strategic management tool, not a documentation checklist.
>
> - Use it to communicate where the firm operates with confidence versus where it is still learning.
> - - Use it to prioritize development work — a Level 1 capability that the firm executes daily is a higher priority than a Level 3 capability rarely used.
>   - - Use it when onboarding new advisors to show them which areas have full support and which require experienced judgment.
>     - - Update it whenever a module reaches a new maturity level. The version milestone and date should be recorded.
