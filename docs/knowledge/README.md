# Knowledge Transfer

> This folder exists because the original objective of the Meritage Operating System was not only to document how Meritage operates — it was to transfer accumulated knowledge from years of working conversations into a form that AI systems can use independently.
> >
> >> Documentation captures the *what*. This folder captures the *why*, the *who*, and the *how we got here*.
> >>
> >> **Classification:** Confidential
> >> **Audience:** Meritage principals and authorized AI systems
> >>
> >> ---
> >>
> >> ## Why This Folder Exists
> >>
> >> The MOS contains excellent documentation of Meritage's advisory methodology. What it does not automatically contain is the institutional memory that accumulated over a year of working conversations — decisions that were considered and rejected, preferences that shaped how documents were written, patterns that emerged from repeated engagement, and the operating model of the person at the center of it all.
> >>
> >> Without this folder, a new AI session starts with the documents but not the context. The context is what makes the collaboration productive.
> >>
> >> This folder is the answer to the question: *What would I tell a new executive before they stepped into this role?*
> >>
> >> ---
> >>
> >> ## What This Folder Is Not
> >>
> >> - It is not a marketing document.
> >> - - It is not a formal SOP.
> >>   - - It is not a substitute for the M&A modules.
> >>     - - It is not a comprehensive biography.
> >>      
> >>       - It is a working knowledge base for AI systems and future advisors who need to understand not just *what Meritage does* but *how Daniel thinks and why Meritage operates the way it does*.
> >>      
> >>       - ---

> >> ## How to Use This Folder
> >>
> >> **For AI systems:** Read this folder before any working session involving Meritage advisory, business development, or strategic decisions. The context here is as important as the process documentation in docs/m-and-a/.
> >>
> >> **For new advisors:** Start with daniels-operating-model.md to understand the philosophy that underlies everything in the MOS. The documents make more sense once you understand the person who built them.
> >>
> >> **For Daniel:** This folder is a living record. When a pattern, preference, or decision feels important enough to preserve, it belongs here. The test is: *Would a capable person who reads this folder be able to make decisions the way I would make them?*

> >> ## A Note on Completeness
> >>
> >> This folder will never be complete. Tacit knowledge accumulates faster than it can be documented. The goal is not completeness — it is sufficiency. The test is whether an AI system reading this folder can operate with the judgment and priorities of someone who has worked with Daniel for a year.
> >>
> >> Entries marked *Pending input from Daniel* require Daniel's direct contribution. They cannot be inferred from documentation alone.
