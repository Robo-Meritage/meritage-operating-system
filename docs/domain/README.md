# Domain Model

> The canonical definitions of every entity, term, and relationship in the Meritage business. Define once. Reference everywhere.
>
> ---
>
> ## What Belongs Here
>
> The domain model is the shared language of MOS. Before documenting processes, workflows, or playbooks, we define the business itself: the objects it works with, the words it uses, and how those objects relate to each other.
>
> This folder is the foundation. Every other document in MOS should link here rather than re-defining terms inline.
>
> ---
>
> ## Documents
>
> | Document | Description |
> |---|---|
> | [glossary.md](glossary.md) | Single canonical definition for every term used across MOS |
> | [business-entities.md](business-entities.md) | Structured definition of every object Meritage works with |
> | [lifecycle.md](lifecycle.md) | The stages a Deal and Engagement move through from first contact to close |
> | [relationships.md](relationships.md) | How entities relate to each other |
>
> ---
>
> ## How to Use This Folder
>
> - **When writing a new document**, check `glossary.md` and `business-entities.md` before defining any term inline. If the term is already defined, link to it.
> - - **When a term is missing**, add it to `glossary.md` first, then reference it in your document.
>   - - **When a process references a lifecycle stage**, link to `lifecycle.md` rather than restating the stage definition.
>     - - **Never redefine a canonical term** in a downstream document. If the definition is wrong, fix it here.
