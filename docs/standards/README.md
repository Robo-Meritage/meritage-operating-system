# docs/standards/

> Cross-cutting quality standards for the Meritage Operating System. Standards that apply to multiple modules live here — not inside any single module.
>
> ---
>
> ## Purpose
>
> This folder contains Meritage Operating Standards — documents that define what "excellent" looks like for a specific discipline or deliverable that spans multiple engagement stages.
>
> Standards documents are distinct from process modules. A module in `docs/m-and-a/` describes how Meritage executes a specific stage of an engagement. A standard in this folder defines the quality bar that applies across stages, across modules, and sometimes across the entire firm.
>
> Every standard here can be referenced by multiple modules and multiple teams, without being redefined in each place.
>
> ## Why Standards Belong Here
>
> Without a dedicated location for cross-cutting standards, there are two failure modes:
>
> 1. **Duplication.** Each module defines its own quality bar in isolation. The valuation standard, the document quality standard, and the communication standard get re-stated with slight variation in each module, creating inconsistency.
>
> 2. 2. **Gaps.** If a standard is "owned" by one module, it is implicitly not the responsibility of the others. A communication standard embedded only in the Buyer Outreach module will not be applied in the Engagement module or the Management Calls module.
>   
>    3. This folder solves both problems. Standards are defined once, owned here, and referenced everywhere they apply.
>   
>    4. ## What Belongs Here
>   
>    5. A document belongs in `docs/standards/` if:
>
>    6. - It defines quality criteria that apply to more than one engagement stage
>       - - It could be used to evaluate whether work product meets the Meritage bar, regardless of which module produced it
>         - - It describes a firm-level expectation rather than a stage-level procedure
>          
>           - A document does **not** belong here if:
>           - - It is a step-by-step process (that belongs in `docs/m-and-a/`)
>             - - It is specific to a single engagement stage only
>               - - It is a template or tool (those belong in the relevant module folder or `docs/templates/`)
